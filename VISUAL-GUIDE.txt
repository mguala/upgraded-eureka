📊 SMACSS CATEGORIES OVERVIEW
═════════════════════════════════════════════════════════════════════════════

    ┌──────────────────────────────────────────┐
    │ SMACSS = 5 Categories + Utilities        │
    └──────────────────────────────────────────┘

    1️⃣  BASE
        └── Default element styles (no classes)
            Examples: h1, p, input, button
            Files: base/

    2️⃣  LAYOUT
        └── Page structure and major sections
            Prefix: l-
            Examples: .l-container, .l-header, .l-grid
            Files: layout/

    3️⃣  MODULE
        └── Reusable components
            No prefix
            Examples: .button, .card, .modal
            Files: module/

    4️⃣  STATE
        └── Changes based on user interaction
            Prefix: is-
            Examples: .is-active, .is-disabled, .is-hidden
            Files: state/

    5️⃣  THEME
        └── Visual configuration (variables only)
            No prefix (variables)
            Examples: $primary-color, $spacing-lg, $font-title
            Files: theme/

    ➕ UTILS
        └── Helper utility classes
            Examples: .text-center, .mt-lg, .flex-center
            Files: utils/


🔀 PROPER IMPORT ORDER (Critical!)
═════════════════════════════════════════════════════════════════════════════

    main-smacss.scss imports in this order:

    1️⃣  Configuration
        @import 'variables';
        @import 'mixins';

    2️⃣  THEME (variables)
        @import 'theme/colors';
        @import 'theme/typography';
        @import 'theme/spacing';

    3️⃣  BASE (default styles)
        @import 'base/reset';
        @import 'base/typography';
        @import 'base/forms';

    4️⃣  LAYOUT (structure)
        @import 'layout/header';
        @import 'layout/navigation';
        @import 'layout/footer';
        @import 'layout/grid';

    5️⃣  MODULE (components)
        @import 'module/button';
        @import 'module/card';
        @import 'module/modal';

    6️⃣  STATE (interactions)
        @import 'state/active';
        @import 'state/disabled';
        @import 'state/hidden';

    7️⃣  UTILS (helpers - most specific)
        @import 'utils/helpers';

    8️⃣  RESPONSIVE (media queries)
        @import 'responsive';

    ✅ This order ensures proper CSS cascade!

🔄 WORKFLOW: FROM SCSS TO CSS
═════════════════════════════════════════════════════════════════════════════

    1. Edit SCSS Files
       assets/scss/*.scss
         │
         ├─ module/_button.scss
         ├─ state/_active.scss
         └─ theme/_colors.scss

    2. Compile with Sass
       sass assets/scss/main-smacss.scss assets/css/main.css
         │
         └─ Generated file: assets/css/main.css

    3. Link in HTML
       <link href="/assets/css/main.css" rel="stylesheet">
         │
         └─ CSS loads in browser

    4. Browser renders styles ✨


🎯 QUICK START (3 STEPS)
═════════════════════════════════════════════════════════════════════════════

        Open index.html and verify styles load ✅


💾 COMPILATION COMMANDS
═════════════════════════════════════════════════════════════════════════════

    Development (watch mode - auto-compile):
    $ sass --watch assets/scss:assets/css

    One-time compilation:
    $ sass assets/scss/main-smacss.scss assets/css/main.css

    Production (minified):
    $ sass --style=compressed assets/scss/main-smacss.scss assets/css/main.css
